# DICOM StoreSCP 现场联调工具软件需求规格说明书

## 1. 文档目的

本文档用于定义 DICOM StoreSCP 现场联调工具的软件需求，包括功能需求、非功能需求、界面需求、异常场景需求和验收要求，作为设计、开发和测试的依据。

## 2. 产品概述

本软件是一款运行于 Windows 电脑上的桌面应用，用于在医院现场替代整机设备完成 DICOM 联调测试，主要支持 CT 向本机推送图像、网络连通性验证及日志取证。

## 3. 用户角色

本软件主要用户包括：

- 研发工程师
- 实施工程师
- 售后工程师
- 测试工程师

## 4. 运行环境

- 操作系统：Windows 10/11
- 网络环境：医院局域网
- 硬件环境：普通笔记本或台式机

## 5. 功能需求

### 5.1 监听配置需求

系统应支持：

- 配置本地 AE Title
- 配置监听端口
- 配置接收目录
- 选择本机监听 IP 或显示本机可用 IP 信息
- 保存和加载上次使用配置

### 5.2 StoreSCP 接收需求

系统应支持：

- 启动 DICOM 监听服务
- 停止 DICOM 监听服务
- 接收来自 CT 的 DICOM 图像
- 保存接收到的图像到本地目录
- 记录来源 AE、来源 IP、接收时间、文件数量、保存结果
- 支持连续接收多个实例

系统应能够：

- 在监听运行中显示当前状态
- 在接收过程中显示实时日志
- 在接收结束后显示本次接收摘要

### 5.3 日志需求

系统应支持记录以下日志：

- 应用启动日志
- 应用停止日志
- 参数配置变更日志
- DICOM 关联建立日志
- DICOM 关联释放日志
- DICOM 关联拒绝日志
- 图像接收成功日志
- 图像接收失败日志
- 文件保存异常日志
- 网络测试日志

系统应支持：

- 按时间查看日志
- 按级别筛选日志
- 导出日志文件

### 5.4 网络调试需求

系统应支持以下网络调试功能：

- Ping 指定 IP
- 检测指定 IP 和端口的连通性
- 对指定 DICOM 节点执行 C-ECHO
- 展示本机网卡和 IP 信息

对于网络测试结果，系统应显示：

- 成功或失败状态
- 响应耗时
- 错误原因或失败信息

### 5.5 接收记录需求

系统应支持：

- 查看最近接收会话记录
- 查看每次会话的开始时间、结束时间、来源 AE、来源 IP、接收数量、状态
- 查看接收文件基本信息

### 5.6 配置管理需求

系统应支持保存以下配置：

- 本地 AE Title
- 监听端口
- 接收目录
- 日志目录
- 目标 C-ECHO 参数
- 是否开启详细日志

## 6. 非功能需求

### 6.1 易用性需求

- 软件应提供图形界面
- 常用操作应在 3 步以内完成
- 界面信息应清晰、直观
- 错误提示应具备可理解性

### 6.2 性能需求

- 软件启动时间应可接受
- 启动监听后应能稳定接收 CT 推送
- 在正常网络条件下，日志显示不应明显阻塞界面操作

### 6.3 可靠性需求

- 异常退出时应尽量保留日志
- 对端口占用、路径不可写、参数非法等场景应有防护
- 停止监听后应释放相关资源

### 6.4 兼容性需求

- 支持 Windows 10/11
- 支持普通医院局域网环境
- 支持主流 CT 设备进行标准 DICOM 推送测试

### 6.5 可维护性需求

- 软件模块之间应低耦合
- 日志格式应统一
- 配置结构应清晰

### 6.6 安全性需求

- 本地接收文件应保存在指定目录
- 涉及患者信息的展示应控制在必要范围
- 软件应为后续日志脱敏和数据清理预留扩展能力

## 7. 界面需求

### 7.1 首页/监听页

应包含：

- 本地 AE Title 输入框
- 监听端口输入框
- 接收目录选择控件
- 本机 IP 显示区域
- 启动监听按钮
- 停止监听按钮
- 当前状态显示区域

### 7.2 网络测试页

应包含：

- Ping 输入与执行按钮
- 端口检测输入与执行按钮
- C-ECHO 参数输入区
- 测试结果显示区

### 7.3 接收记录页

应包含：

- 接收会话列表
- 接收摘要显示区
- 接收文件信息查看区

### 7.4 日志页

应包含：

- 实时日志列表
- 日志级别筛选
- 日志搜索
- 日志导出按钮

## 8. 异常场景需求

系统应考虑以下异常场景：

- 监听端口已被占用
- 本地 AE Title 为空
- 接收目录不存在或不可写
- CT 无法连通本机
- C-ECHO 失败
- DICOM Association 被拒绝
- 图像接收中断
- 文件保存失败
- 本机多网卡导致监听 IP 选择错误

对于上述场景，系统应：

- 提示明确错误信息
- 在日志中记录详细上下文
- 尽量给出建议排查方向

## 9. 验收需求

项目验收时，至少应满足以下要求：

### 9.1 功能验收

- 可成功启动和停止监听
- 可接收 CT 推送的 DICOM 图像
- 接收图像可正确落盘
- 可执行 Ping
- 可执行端口检测
- 可执行 C-ECHO
- 可查看和导出日志

### 9.2 稳定性验收

- 多次启动与停止监听无明显异常
- 正常接收过程中程序不崩溃
- 异常场景下程序可给出提示并记录日志

### 9.3 可用性验收

- 工程师可通过界面独立完成监听配置和测试操作
- 常用联调流程清晰可执行

## 10. 后续扩展需求

后续版本可考虑增加：

- 接收记录持久化管理
- 测试报告导出
- DICOM 标签查看
- 基础图像预览
- 脱敏导出
- StoreSCU 反向发送测试

## 11. 结论

本文档定义了 DICOM StoreSCP 现场联调工具的基础需求范围。项目开发应优先满足 StoreSCP 接收、日志、网络调试三项核心需求，并以 Windows 桌面应用形式交付，确保产品具备现场可用性和问题定位能力。
