最后更新时间：2026-04-04  
适用版本：当前标准发布目录 `dist\win-x64`  
文档类型：交付文档

## 1. 文档目的

本文档用于说明当前版本如何以“绿色版”方式交付和使用。

这里的“绿色版”指：

- 不依赖目标电脑预装 `.NET SDK`
- 不需要复杂安装流程
- 通过复制整个发布目录即可运行

## 3. 交付方式

### 3.1 正确交付方式

应交付整个目录：

- [dist\win-x64](C:\Users\wik\Desktop\storescp工具\dist\win-x64)

建议做法：

1. 将整个 `dist\win-x64` 文件夹打包为 zip
2. 发送给目标使用者
3. 对方解压到本地任意可写目录后运行

### 3.2 不正确交付方式

不要只单独发送：

- [StorescpTool.App.exe](C:\Users\wik\Desktop\storescp工具\dist\win-x64\StorescpTool.App.exe)

原因：

- 当前不是单文件发布
- 程序运行依赖同目录下的大量 DLL 和运行时文件

## 4. 目标电脑要求

建议目标电脑满足：

- `64 位 Windows`
- 建议 `Windows 10` 或 `Windows 11`
- 有本地文件写入权限
- 允许监听测试端口

补充说明：

- 当前发布目标是 `win-x64`
- 不建议用于 `32 位 Windows`
- 是否能在“任意 Windows 电脑”运行，仍受系统版本、权限、杀毒软件、组策略、防火墙影响

## 5. 建议解压位置

建议解压到普通可写目录，例如：

```text

```

或：

```text
D:\StorescpTool\
```

不建议放在权限较严格或经常被系统保护的目录，例如：

```text

```

首次运行时，程序会在应用目录下自动生成运行时目录和文件，例如：


## 7. 当前默认参数

当前默认值为：

- `本地 AE Title = RC120`
- `监听端口 = 5678`
- `监听 IP = 自动选择本机优先 IPv4`
- `C-ECHO Calling AE = RC120`
- `C-ECHO Called AE = CT_AE`

注意：

- 首次启动时会使用以上默认值
- 一旦生成 `config\appsettings.json`，后续将优先加载该配置文件，而不是重新采用代码默认值

## 8. 现场使用建议

建议首次使用按以下顺序操作：

1. 启动程序
2. 检查 `AE Title`、监听 IP、监听端口
3. 点击“启动监听”
4. 先做 `Ping`
5. 再做 TCP 端口检测
6. 如有需要，做 `C-ECHO`
7. 如需自检，执行本机 `C-STORE` 自测
8. 再让 CT 或模拟器向该电脑推送图像

## 10. 常见问题

### 10.1 双击后程序打不开

可能原因：

- 文件未完整拷贝
- 只拿了 exe，没有带依赖文件
- 被杀毒软件或系统策略拦截

建议检查：

- 是否解压了完整目录
- 是否是从 zip 中直接运行
- 是否被 Windows SmartScreen 拦截

### 10.2 程序能开，但监听失败

可能原因：

- 端口已被占用
- 本机防火墙阻止
- 监听 IP 不正确

建议检查：

- 更换端口
- 检查防火墙入站规则
- 重新刷新监听 IP 并选择正确网卡地址

### 10.3 CT 无法推送到该电脑

可能原因：

- IP 不通
- 端口不通
- AE Title 不匹配
- CT 目标节点配置错误

建议排查顺序：

1. `Ping`
2. TCP 端口检测
3. `C-ECHO`
4. 检查 CT 上配置的目标 AE / IP / Port

### 10.4 默认参数不对

可能原因：

- 程序目录下已经存在旧的：

```text
config\appsettings.json
```

说明：

- 运行时配置会覆盖代码默认值

处理方式：

1. 修改程序界面中的配置并保存
2. 或删除旧配置后重新启动

## 11. 绿色版升级建议

3. 如需保留历史配置和日志，再将旧的 `config`、`data` 合并回去
