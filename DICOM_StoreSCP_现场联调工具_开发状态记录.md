## 4. 当前可用能力

截至当前版本，工具已经具备以下可运行能力：

- 启动桌面界面
- 配置本地 AE Title、监听 IP、监听端口、接收目录、日志目录
- 执行 Ping
- 执行 TCP 端口检测
- 执行 DICOM C-ECHO
- 执行本机 C-STORE 自测
- 启动和停止 StoreSCP
- 将日志写入界面和本地日志文件
- 展示接收到的 DICOM 记录
- 展示接收会话统计
- 导出诊断包
- 程序重启后保留接收记录和会话统计
- 对关键输入做基础校验并在界面提示
- 可通过发布脚本生成 `dist\win-x64` 交付目录

## 6. 当前风险

### 6.1 DICOM 现场兼容性风险

虽然 StoreSCP 骨架已可编译并具备接收逻辑，但仍需使用真实设备或 DICOM 模拟工具验证不同 CT 的传输语法、Association 协商和异常场景。

### 6.2 界面可用性仍可增强

当前界面已可用，但还缺少目录选择器、输入校验、日志检索等工程化能力。

## 7. 当前版本结论

当前项目已完成从 0 到 1 的基础落地，具备继续向“现场可用 MVP”推进的条件。下一步应优先围绕 StoreSCP 联调闭环、接收记录增强、日志增强和发布准备继续推进。
