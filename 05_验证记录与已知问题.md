## 4. 已知运行行为

### 4.1 运行时配置会覆盖代码默认值

当前默认值在：

- [AppConfig.cs](C:\Users\wik\Desktop\storescp工具\src\StorescpTool.Core\Models\AppConfig.cs)
- [EchoSettings.cs](C:\Users\wik\Desktop\storescp工具\src\StorescpTool.Core\Models\EchoSettings.cs)

但一旦运行目录下生成：

```text
config\appsettings.json
```

程序后续将优先加载该配置，而不是重新采用代码默认值。

### 4.2 正在运行时发布到同目录可能失败

如果 `StorescpTool.App.exe` 正在运行，直接发布到同一个输出目录可能出现文件锁定错误。

建议做法：

1. 关闭正在运行的程序
2. 再发布

### 4.3 当前 shell 里可能没有 dotnet PATH

建议统一使用完整路径：

```powershell
& 'C:\Program Files\dotnet\dotnet.exe' ...
```

### 4.4 当前目录已接入 Git

当前目录已经初始化 Git，并已推送远端：

- 默认分支：`main`
- 远端仓库：`git@github.com:kuangliu774392519/storeSCP_tool.git`

### 4.5 现场同事不应从源码仓库直接运行

当前推荐方式：

- 维护者维护源码仓库
- 现场同事从 GitHub Releases 下载绿色版 zip

不推荐：

- `git clone` 后直接运行
- 从 GitHub 源码 ZIP 中查找 exe

### 4.6 PowerShell 里部分中文显示可能乱码

在某些命令输出中，中文可能因为终端编码显示异常；目前这更像是终端显示问题，不影响源码编译和程序运行。
