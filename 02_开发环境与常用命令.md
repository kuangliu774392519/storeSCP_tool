## 3. 本地发布命令

标准发布到 `dist\win-x64`：

```powershell
.\scripts\Publish-StorescpTool.ps1 -Configuration Release -Runtime win-x64
```

等价命令：

```powershell
& 'C:\Program Files\dotnet\dotnet.exe' publish '.\src\StorescpTool.App\StorescpTool.App.csproj' -c Release -r win-x64 --self-contained true -p:PublishSingleFile=false -p:IncludeNativeLibrariesForSelfExtract=true -o '.\dist\win-x64'
```

## 4. 本地绿色包命令

生成可上传到 GitHub Releases 的绿色版 zip：

```powershell
.\scripts\Publish-StorescpTool.ps1 -Configuration Release -Runtime win-x64 -CreatePortableZip -PackageVersion 0.1.0
```

默认输出：

- 发布目录：`dist\win-x64`
- 绿色包目录：`.artifacts\portable`

生成物命名规则：

```text
StorescpTool_win-x64_portable_<version>.zip
StorescpTool_win-x64_portable_<version>.sha256.txt
```

## 5. GitHub Release 发版命令

推荐正式发版方式：

```powershell

```

- GitHub Actions 会自动执行测试

## 8. 运行时配置路径

运行时配置由 [AppConfigService.cs](C:\Users\wik\Desktop\storescp工具\src\StorescpTool.Infrastructure\Config\AppConfigService.cs) 管理，真实配置文件路径是：

```text
<应用目录>\config\appsettings.json
```

重要行为：

- 如果该文件不存在，程序会按 [AppConfig.cs](C:\Users\wik\Desktop\storescp工具\src\StorescpTool.Core\Models\AppConfig.cs) 默认值创建
- 如果该文件存在，它会覆盖代码默认值

## 10. 发布与打包时常见问题

### 10.1 文件被锁

如果当前程序正在运行，发布到同一个输出目录时可能出现 DLL 或 EXE 被锁。

处理方式：

1. 先关闭正在运行的 `StorescpTool.App.exe`
2. 再执行标准发布

### 10.2 PATH 中没有 dotnet

处理方式：

- 使用完整路径 `C:\Program Files\dotnet\dotnet.exe`
